# dokploy_redirect (Resource)

Manages URL redirects and rewrites for a Dokploy application. Redirects are useful for:
- Redirecting www to non-www (or vice versa)
- Forcing HTTPS connections
- Handling legacy URLs
- Domain migrations
- URL canonicalization

## Example Usage

### Redirect www to non-www

```terraform
resource "dokploy_redirect" "www_to_non_www" {
  application_id = dokploy_application.myapp.id
  regex          = "^https://www\\.example\\.com/(.*)"
  replacement    = "https://example.com/$1"
  permanent      = true
}
```

### Redirect HTTP to HTTPS

```terraform
resource "dokploy_redirect" "http_to_https" {
  application_id = dokploy_application.myapp.id
  regex          = "^http://(.+)"
  replacement    = "https://$1"
  permanent      = true
}
```

### Redirect Old Blog Path

```terraform
resource "dokploy_redirect" "old_blog" {
  application_id = dokploy_application.myapp.id
  regex          = "^https://example\\.com/old-blog/(.*)"
  replacement    = "https://example.com/blog/$1"
  permanent      = false
}
```

### Redirect Legacy API Endpoint

```terraform
resource "dokploy_redirect" "legacy_api" {
  application_id = dokploy_application.api.id
  regex          = "^https://api\\.example\\.com/v1/(.*)"
  replacement    = "https://api.example.com/v2/$1"
  permanent      = true
}
```

### Redirect Subdomain to Main Domain

```terraform
resource "dokploy_redirect" "subdomain" {
  application_id = dokploy_application.myapp.id
  regex          = "^https://old\\.example\\.com/(.*)"
  replacement    = "https://example.com/$1"
  permanent      = true
}
```

## Schema

### Required

- `application_id` (String) The ID of the application.
- `regex` (String) Regular expression to match the URL.
- `replacement` (String) Replacement URL pattern.

### Optional

- `permanent` (Boolean) Whether the redirect is permanent (301) or temporary (302). Defaults to `true`.

### Read-Only

- `id` (String) The unique identifier of the redirect.

## Redirect Types

- **Permanent (301)**: Use `permanent = true` for SEO-friendly permanent redirects. Search engines will update their indexes.
- **Temporary (302)**: Use `permanent = false` for temporary redirects that may change in the future.

## Regex Patterns

The `regex` field supports standard regular expressions. Common patterns:

- `^https://www\.example\.com/(.*)` - Match www subdomain
- `^http://(.+)` - Match any HTTP URL
- `^https://example\.com/old-path/(.*)` - Match specific path
- `^https://old\.example\.com/(.*)` - Match specific subdomain

The `replacement` field can reference captured groups using `$1`, `$2`, etc.
